# CROSSCHECK — ISA model vs. an authoritative RISC-V spec (task #7)

This is the plan for the **last open item** in the hex0 verification tower: removing
*"our hand-rolled RV64I model is faithful to real RISC-V"* from the TCB
(`TCB.md` item 1) by cross-checking `Rv64i.{v,lean}`'s decode+step against an
authoritative, independently-maintained RISC-V semantics.

Read with `STATUS.md` (state), `TCB.md` (trust boundary), `PROOF.md` (refinement
methodology). Everything *above* the ISA layer (`core_refines`, grammar,
certification) is already proved in both Lean and Coq; this is the floor of the
tower.

## 1. The deliverable: two theorems

New file `coq/RvCross.v` (logical name `Hex0Coq.RvCross`), added to `_CoqProject`
after `Rv64i.v`/`Image.v`.

### T1 — Decode agreement (pure; the high-value, self-contained half)

> For every 32-bit word `w`, whenever our decoder returns a real instruction, the
> riscv-coq decoder (at `RV64I`) returns the corresponding instruction with
> identical operands; and on the encodings our decoder rejects within the opcodes
> we model, riscv-coq also yields no I-instruction.

```coq
Definition embed (i : Rv64i.Instr) : Decode.Instruction := (* 12-arm map, §4 *)
Theorem decode_agrees : forall w, 0 <= w < 2^32 ->
  forall i, Rv64i.decode w = i -> i <> Iunknown ->
    Decode.decode RV64I w = embed i.
```

This removes "our **decoder** is wrong" from the TCB unconditionally (all inputs),
modulo the one **intentional narrowing** in §5 (SLLI `shamt ≥ 32`).

### T2 — Execute / step agreement (state-bridge; over the 12 forms)

> Under a bridge relation `R` between our `State` and riscv-coq's `RiscvMachine`,
> and the well-formedness preconditions in §3, one of our `step`s corresponds to
> one riscv-coq instruction cycle (`Run.run1 RV64I`), which succeeds and lands in a
> related state.

```coq
Theorem step_agrees : forall s m,
  R s m -> WF s m ->
  exists m', Run.run1 RV64I m = (Some tt, m') /\ R (Rv64i.step s) m'.
```

(`Run.run1` over the **Minimal** machine's `OState RiscvMachine` monad — see §2.)
T2 removes "our **step** is wrong" from the TCB for the 12 forms, under the
preconditions that hold for the loaded `core` (§5 shows each precondition is
discharged by the real image / `WellFormed`).

T1 is independently valuable and should land first; T2 builds on T1 (so the
decode inside `run1` is already pinned).

### PC convention (critical for T2)

`execute` of a **branch/jump** calls `setPC newPC`, which on Minimal is
`update (withNextPc newPC)` — it writes **`getNextPc`, not `getPc`**.
Non-control instrs never call `setPC`. Then `endCycleNormal` does
`pc := nextPc; nextPc := nextPc + 4`. So the machine maintains the invariant
**`getNextPc = getPc + 4` at the start of every `run1`**, and:

- sequential instr: `nextPc` untouched by execute ⇒ after cycle `pc' = pc+4`. ✓ matches our `wadd pc 4`.
- taken branch to `T`: `setPC T` ⇒ after cycle `pc' = T`. ✓ matches our `wadd pc imm`.

⇒ The bridge `R` must include `m.(getNextPc) = m.(getPc) + 4` as part of
well-formedness, and T2's `R (step s) m'` re-establishes it.

### Fetch & memory caveats (become preconditions)

- `loadWord Fetch pc` calls `loadN 4` which **fails (`fail_hard`)** unless
  `isXAddr4B pc (getXAddrs)` — the 4 fetched bytes must be marked *executable*.
  ⇒ precond: code region ⊆ `getXAddrs`. (`core` is loaded executable; trivially true.)
- `loadN`/`storeN` fail unless the address range is present in the `Mem` map.
  ⇒ precond: the touched addresses are in `getMem`'s domain.
- `getRegister`/`setRegister` `fail_hard` unless `reg = 0` or `0 < reg < 32`.
  Our decoded register indices are `field _ _ 5 ∈ [0,32)` by construction — discharge
  with a `field` range lemma. (`x0` reads 0 / ignores writes on **both** sides. ✓)
- `combine 4 inst` little-endian-combines the 4 fetched bytes to a `Z` — matches our
  `fetch32` (`b0 + b1·256 + b2·2^16 + b3·2^24`). Prove `combine 4 (load_bytes …) = fetch32`.
- `translate Load/Store size a` (in `ExecuteI` for `Lbu`/`Sb`) is the identity on the
  Minimal machine for our accesses (no MMU; size-1 access always aligned). **Confirm**
  `Spec.Machine.translate`'s Minimal definition reduces to `Return a` for size 1.

## 3. The state bridge `R : State -> RiscvMachine -> Prop`

Our Coq `State = { reg : Z->Z ; pc : Z ; mem : Z->Z }` (total functions, `Z`).
riscv-coq state uses `coqutil` finite maps and `word 64`. The relation, over the
region actually exercised:

```
R s m :=
  (* PC *)         word.unsigned m.(getPc) = s.(pc)
  (* nextPc inv *) word.unsigned m.(getNextPc) = wrap (s.(pc) + 4)
  (* regs *)       forall r, 0 < r < 32 ->
                     word.unsigned (reg_get m r) = s.(reg) r            (* r=0 ⇒ both 0 *)
  (* mem *)        forall a, a ∈ Dom -> map.get m.(getMem) (word.of_Z a) = Some (byte.of_Z (s.(mem) a))
                   /\ 0 <= s.(mem) a < 256
  (* exec *)       CodeRegion ⊆ word-image of m.(getXAddrs)
```

`WF s m` (well-formedness preconditions, all true for the loaded `core`):
`getNextPc = getPc+4`; fetch addr + its 3 successors ∈ `Mem` ∩ `XAddrs`; any
load/store target ∈ `Mem`; branch/jump targets are 4-aligned (§5).

`Dom` = code ∪ input ∪ output regions (the `WellFormed` layout from `PROOF.md §3`:
code `[0x88,0x1cc)`, input `[0x1cc, …)`, output `[0x3b0, …)`). For T2 we only need
`R` preserved on `Dom`; the `sb` store stays within output (already proved disjoint
in `Refine.v`'s `store_epilogue`/frame lemmas — reuse that geometry).

## 4. Per-instruction mapping (the `embed` table + execute correspondence)

| our `Instr` (Rv64i.v) | riscv-coq `InstructionI` | our `step` effect | riscv-coq `execute`+`run1` effect | agree because |
|---|---|---|---|---|
| `Iaddi rd rs1 imm` | `Addi rd rs1 imm` | `rd := wadd (rget rs1) imm`; pc+=4 | `setRegister rd (x + fromImm imm)` | `x+fromImm` = word add = `wadd`; imm both `sext 12` |
| `Iadd rd rs1 rs2` | `Add rd rs1 rs2` | `rd := wadd r1 r2` | `setRegister rd (x+y)` | word add |
| `Ior rd rs1 rs2` | `Or rd rs1 rs2` | `rd := Z.lor r1 r2` | `setRegister rd (or x y)` | `or` = bitwise = `wor` (in range, no wrap) |
| `Islli rd rs1 sh` | `Slli rd rs1 sh` | `rd := wrap (shiftl r1 sh)` | `setRegister rd (sll x sh)` | `sll` = `wrap∘shiftl`; **see §5 SLLI** |
| `Ilbu rd rs1 imm` | `Lbu rd rs1 imm` | `rd := mem(wadd r1 imm) mod 256` | `loadByte (translate (a+imm)); uInt8ToReg` | `uInt8ToReg` zero-extends; `translate`=id (§2) |
| `Isb rs1 rs2 imm` | `Sb rs1 rs2 imm` | `mem[wadd r1 imm] := r2 mod 256` | `storeByte (translate (a+imm)) (regToInt8 r2)` | `regToInt8` = low byte = `mod 256` |
| `Ibeq rs1 rs2 imm` | `Beq rs1 rs2 imm` | `pc := (r1=?r2)? wadd pc imm : pc+4` | `when (x=y) setPC(pc+imm)`; else fallthrough | `reg_eqb` on words ⇔ `Z.eqb` on `unsigned`; **align §5** |
| `Iblt rs1 rs2 imm` | `Blt rs1 rs2 imm` | `pc := sltb r1 r2 ? …` | `when (x < y) …` (`<` = **signed** on `word`) | `sltb` = signed `<` = MachineWidth `<` |
| `Ibge rs1 rs2 imm` | `Bge rs1 rs2 imm` | `pc := sltb r1 r2 ? pc+4 : wadd pc imm` | `when (x >= y) …` | `>=` = `negb signed<` = `negb sltb` |
| `Ibgeu rs1 rs2 imm` | `Bgeu rs1 rs2 imm` | `pc := ultb r1 r2 ? pc+4 : wadd pc imm` | `when (negb (ltu x y)) …` | `ultb` = `ltu` (unsigned `<`) |
| `Ijal rd imm` | `Jal rd imm` | `rd := pc+4; pc := wadd pc imm` | `getPC; setReg rd (pc+4); setPC (pc+imm)` | direct; **align §5** |
| `Ijalr rd rs1 imm` | `Jalr rd rs1 imm` | `t := a - a mod 2`; `rd:=pc+4; pc:=t` (`a=wadd r1 imm`) | `and (x+imm) (lnot 1)`; setReg; setPC | `and _ (lnot 1)` = clear bit0 = `a - a mod 2`; **align §5** |

`embed` is the obvious 12-arm constructor map (already written & checked in the
empirical probe). The immediate columns are **equal `Z` values**, not just
"corresponding", because both sides sign-extend identically — that is what makes
T1 a clean equality.

## 5. Differences the cross-check **surfaces** (the payoff)

These are exactly the facts that were previously hidden in "trust the model".
Each is either (a) an intentional, documented narrowing, or (b) a precondition
that provably holds for the loaded `core`.

1. **SLLI `shamt ≥ 32` (intentional narrowing).** Our decoder requires
   `funct7 = 0` (bits 25..31 = 0), so it only accepts `shamt ∈ [0,31]` and returns
   `Iunknown` for `shamt ∈ [32,63]`. riscv-coq at `RV64I` accepts the full 6-bit
   `shamt ∈ [0,63]` (`shamtHiTest` allows bit 25 when `bitwidth=64`). ⇒ T1 holds
   **for the encodings our decoder accepts** (which include *every* `core` SLLI —
   `slli …,4`, `shamt=4`). Document as a deliberate restriction of the modelled
   ISA subset; it is sound (we never *mis*-decode, we only decline). On accepted
   encodings (`shamt<32`) `sll x shamt = wrap(shiftl x shamt)` = our `wshl`.

2. **Misaligned branch/jump target → exception (precondition).** riscv-coq's
   `Beq/Blt/Bge/Bgeu/Jal/Jalr` compute `newPC` and, if `remu newPC 4 ≠ 0`, call
   `raiseExceptionWithInfo` (instruction-address-misaligned) instead of jumping; on
   Minimal that's `fail_hard`. Our model unconditionally sets `pc`. ⇒ T2 carries the
   precondition **"the taken target is 4-aligned"**. For `core` every branch/jump
   target is the address of a 4-byte-aligned instruction, so the exception is never
   taken and the two agree. (This is genuinely *new* information the cross-check
   extracts: our model is only faithful for aligned targets — true here, but now
   explicit rather than assumed.)

3. **JALR low-bit clear (agree).** Ours `a - a mod 2`; theirs `and (a) (lnot 1)`.
   Equal for all `a`. (riscv-coq additionally alignment-checks the *result*; for
   `ret`→0 it's aligned. Falls under (2).)

4. **Fetch executability & memory-domain (preconditions).** `loadWord Fetch`
   needs `isXAddr4B`; `loadN/storeN` need the address mapped. Our total-function
   memory has no such notion. ⇒ preconditions in `WF`, discharged by the loaded
   image (code executable; `Dom` mapped).

5. **Register-index validity (discharged, not assumed).** riscv-coq `fail_hard`s on
   `reg ≥ 32`; our `field _ _ 5` is always `< 32`. A one-line `field` bound closes it.

6. **x0 semantics (agree, no obligation).** Both read x0 as 0 and ignore writes
   (`rget/rset` vs `Register0` guards).

None of these is a *bug* in our model relative to the real hardware — they are the
precise boundary of the modelled subset, now made explicit. That is the whole point
of task #7.

## 6. Lemma DAG / proof structure (mirrors `PROOF.md §5` style)

```
T1: decode_agrees
  field_range            field w lo 5 ∈ [0,32); field w lo 7 ∈ [0,128)   (Z arith)
  imm_eq_*               our sext k = riscv signExtend k (Beq/Jal/etc.)   (Z bit-ident)
  decode_case_<op>       one per opcode group: unfold both decoders,
                           Z.eqb opcode dispatch, vm_compute the riscv side,
                           reflect field equalities                       (12-ish cases)
  decode_agrees          assemble case split on opcode/funct3/funct7

T2: step_agrees   (depends on T1 for the fetch/decode segment)
  word_bridge            unsigned/of_Z round-trips on Dom; word.add=wadd,
                           word ltu/lts = ultb/sltb, sll/and/or lemmas     (coqutil)
  combine_fetch          combine 4 (load_bytes 4 mem pc) = fetch32 s       (LE bytes)
  reg_bridge             getRegister r in OState = Return (of_Z (rget r))  (map.get)
  mem_bridge             loadByte/storeByte ↔ our mem on Dom               (map.get/put)
  run1_unfold            run1 RV64I m = bind getPC … endCycleNormal        (OState calc)
  exec_<instr>           12 lemmas: execute (embed i) on m steps R-related (uses §4 row)
  step_agrees            case on decode (via T1), dispatch to exec_<instr>,
                           endCycleNormal re-establishes nextPc invariant
```

The `exec_<instr>` proofs are uniform: `unfold run1, execute, the Minimal class
methods; simpl the OState binds; rewrite reg/mem/word bridges; reflect the guard
(branch taken?) with the `Z.eqb`/`ltu`/`lts` bridge; reconstruct `R`.` Heaviest
machinery is the `coqutil` `word`/`map` lemmas (`Word.Properties`,
`Map.Properties`) — all installed.

> Our decode (all 32-bit words, modulo the SLLI `shamt<32` subset) and our step

> artifact than our 136-line model.

i.e. we trade "trust our tiny bespoke model" for "trust the community's reference
model", which is the standard and much stronger position.
